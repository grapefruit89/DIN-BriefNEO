# Contributing to DIN 5008 Generator

> **Role:** Human Developer Guide.
> **Philosophy:** Brain-First, Code-Second.

## 🛑 STOP! Read Before Coding

This project uses a strict **Traceability Architecture**.
You cannot just "edit the code".

### The Workflow

1.  **Check the Brain**: Look into `.brain/` folder.
    - Is the feature defined in `02_business_logic.md`?
    - Is the UI defined in `04_ui_design.md`?
2.  **Update the Brain**:
    - If you add a feature, add a Spec-ID (e.g., `[DIN-FEAT-NEW]`) to the brain first.
    - Update `06_roadmap_todo.md`.
3.  **Code**:
    - Implement changes.
    - **MUST**: Add the Spec-ID as a comment near the code block.
    - Example: `// [DIN-FEAT-NEW] Implemented logic here`
4.  **Verify**:
    - Run `App.Tests.run()` in the browser console (or via UI).

### Project Structure

- **.brain/**: The Boss. Source of Truth.
- **js/core/**: State & Config.
- **js/logic/**: Pure functions (easier to test).
- **js/ui/**: DOM manipulation (harder to test).

### Testing

Run the in-browser test suite via the "System Diagnostics" dialog or strictly in console:
~~~javascript
await window.App.Tests.run();
~~~

 * > **File:** js/services/api.js
 * > **Role:** External Service Gateway.

window.App = window.App || {};

window.App.API = {
    currentController: null,

    /**
     * @param {string} query 
     * @param {object} config 
     * @param {object|null} bias { lat, lon }
     */
    fetchAddressSuggestions: async function(query, config, bias = null) {
        try {
            if (query.length < 3) return [];

            if (this.currentController) this.currentController.abort();
            this.currentController = new AbortController();
            const signal = this.currentController.signal;

            let url = '';
            if (config.provider === 'photon') {
                url = `https://photon.komoot.io/api/?q=${encodeURIComponent(query)}&lang=de&limit=5`;
                if (bias) url += `&lat=${bias.lat}&lon=${bias.lon}`;
            } else if (config.provider === 'geoapify' && config.apiKey) {
                url = `https://api.geoapify.com/v1/geocode/autocomplete?text=${encodeURIComponent(query)}&apiKey=${config.apiKey}&limit=5&lang=de`;
                if (bias) url += `&bias=proximity:${bias.lon},${bias.lat}`;
            } else {
                return [];
            }

            // [AUDIT-FIX-02] Use Helper for Backoff
            const res = await this._fetchWithRetry(url, { signal }, 3, 300);
            
            if (!res.ok) throw new Error(`HTTP Error: ${res.status}`);
            const data = await res.json();
            return this.parseFeatures(data.features, config.provider);

        } catch (err) {
            if (err.name === 'AbortError') return [];
            window.App.Logger.error('API', `Suggestion Fetch Failed: ${err.message}`);
            return [];
        }
    },

    /**
     * [AUDIT-FIX-02] Private Helper: Fetch with Exponential Backoff
     * @param {string} url
     * @param {object} options
     * @param {number} retries
     * @param {number} backoff (ms)
     */
    _fetchWithRetry: async function(url, options = {}, retries = 3, backoff = 300) {
        for (let i = 0; i < retries; i++) {
            try {
                const res = await fetch(url, options);
                // Return immediately if successful or if client error (4xx) which shouldn't be retried
                if (res.ok || (res.status >= 400 && res.status < 500)) {
                    return res;
                }
                // If 5xx, throw to trigger retry
                throw new Error(`Server Error: ${res.status}`);
            } catch (err) {
                // Don't retry aborts
                if (err.name === 'AbortError') throw err; 
                // Don't retry if it's the last attempt
                if (i === retries - 1) throw err;

                // Exponential Backoff: 300ms, 600ms, 1200ms...
                const delay = backoff * Math.pow(2, i);
                if (window.App.Logger) window.App.Logger.warn('API', `Retry ${i+1}/${retries} in ${delay}ms...`);
                await window.App.Utils.sleep(delay);
            }
        }
    },

    /**
     * Converts a city/zip string into coordinates for the profile anchor.
     * [DIN-FEAT-RADIUS-HYBRID]
     */
    geocode: async function(addressString) {
        try {
            if (!addressString) return null;
            const url = `https://photon.komoot.io/api/?q=${encodeURIComponent(addressString)}&limit=1`;
            // Use simple fetch for one-off geocoding, or enhance with retry if critical.
            // Using standard fetch here as it's less time-sensitive than autocomplete.
            const res = await fetch(url);
            if (!res.ok) throw new Error(`HTTP Error: ${res.status}`);
            const data = await res.json();
            if (data.features && data.features.length > 0) {
                const [lon, lat] = data.features[0].geometry.coordinates;
                return { lat, lon };
            }
        } catch (e) {
            window.App.Logger.error('API', `Geocoding Failed: ${e.message}`);
            return null;
        }
        return null;
    },

    parseFeatures: function(features, provider) {
        try {
            if (!features || !features.length) return [];
            return features.map(f => {
                const p = f.properties;
                let display = '';
                let insertion = '';

                if (provider === 'photon') {
                    display = `${p.name || ''} ${p.street || ''} ${p.housenumber || ''}, ${p.city || ''}`.replace(/\s+/g, ' ').trim();
                    const street = `${p.street || ''} ${p.housenumber || ''}`.trim();
                    const city = `${p.postcode || ''} ${p.city || ''}`.trim();
                    insertion = [p.name || '', street, city].filter(Boolean).join('\n');
                } else {
                    display = p.formatted;
                    insertion = `${p.address_line1}\n${p.address_line2}`;
                }
                return { display, insertion, type: 'api' };
            });
        } catch (e) {
            window.App.Logger.error('API', `Parsing Features Failed: ${e.message}`);
            return [];
        }
    }
};

    /**
     * Derives the single-line return address using the "Anchor Strategy".
     * @see [DIN-LOGIC-RET-ANCHOR]
     */
    deriveReturnLine: function(rawSenderText) {
        let text = rawSenderText.trim();
        if (!text) return '';
        const lines = text.split(/[\n,]+/).map(p => p.trim()).filter(p => p);
        if (lines.length === 0) return '';
        const zipRegex = /(^|\s)\d{5}(\s|$)/;
        const anchorIndex = lines.findIndex(l => zipRegex.test(l));

        let namePart = '', streetPart = '', cityPart = '';

        if (anchorIndex !== -1) {
            cityPart = lines[anchorIndex];
            if (anchorIndex > 0) streetPart = lines[anchorIndex - 1];
            if (anchorIndex > 1) namePart = lines[0]; 
            else if (anchorIndex === 1) namePart = lines[0];
        } else {
            if (lines.length > 0) namePart = lines[0];
            if (lines.length > 1) streetPart = lines[1];
        }
        const formattedName = this._formatSenderName(namePart);
        return [formattedName, streetPart, cityPart].filter(Boolean).join(', ');

    analyzeAddressQuality: function(text, type = 'recipient') {

        if (!text || text.trim().length < 5) return issues;

        const lines = text.split('\n').map(l => l.trim()).filter(l => l);
        const fullText = lines.join(' ');

        // 1. ZIP Code Check (DACH: 4-5 digits)
        const zipMatch = fullText.match(/\b\d{4,5}\b/);
        if (!zipMatch) {
            issues.push("PLZ fehlt (4-5 Stellen)");
        }

        // Pattern: Word(s) followed by space and at least one digit
        const hasStreetNum = lines.some(l => /\b[a-zA-ZäöüÄÖÜß.\-\s]+\s+\d+[a-zA-Z]?\b/.test(l));
        if (!hasStreetNum) {
            issues.push("Straße/Nr. unklar");

        if (type === 'sender') {
            const hasEmail = fullText.includes('@');

        const titleRegex = /(Prof\.|Dr\.|Dipl\.-[A-Za-z]+|Mag\.)/g;
        
        for (const line of lines) {
            const clean = line.trim();
            if (!clean) continue;

            // 1. Check for specific addressing keywords
            if (/(^|\s)(Herr|Herrn)(\s|$)/i.test(clean)) {
                gender = 'm';
            } else if (/(^|\s)(Frau)(\s|$)/i.test(clean)) {
                gender = 'f';
            } else if (/(^|\s)(Familie|Eheleute)(\s|$)/i.test(clean)) {
                gender = 'fam';
            }

            // If we found a gender marker on this line, try to extract the name from THIS line
            if (gender !== 'n' && !name) {
                // Remove addressing prefix
                let content = clean.replace(/^(Herr|Herrn|Frau|Familie|Eheleute)\s+/i, '');
                
                // Extract Titles
                const tMatches = content.match(titleRegex);
                if (tMatches) {
                    titlesFound = tMatches.join(' ') + ' ';
                }
                
                // Remove titles from content to get raw name
                content = content.replace(titleRegex, '').trim();
                
                // Simple First/Last Split
                const parts = content.split(/\s+/);
                if (parts.length > 0) {
                    // Assume last word is last name
                    name = parts[parts.length - 1]; 
                    // Everything else is potentially first name
                    if (parts.length > 1) {
                        firstName = parts.slice(0, parts.length - 1).join(' ');
                    }
                }
                
                // Break after finding the primary addressee line
                break;
            }
        }
        return { gender, name, title: titlesFound.trim(), firstName };
    },

    /**
     * Smart Salutation Derivation [DIN-LOGIC-GREETING-2026]
     */
    deriveSalutation: function(analysis, formality) {
        const { gender, name, title, firstName } = analysis;
        const P = window.App.Config.PHRASES;

        // Unknown Gender (Company or ambiguous) -> Default
        if (gender === 'n' || !name) return P.SALUTATION_DEFAULT; // "Sehr geehrte Damen und Herren,"
        
        // Multi-Person
        if (gender === 'fam') return `Sehr geehrte Familie ${name},`;

        // CASUAL: Try First Name -> Last Name
        if (formality === 'casual') {
            if (firstName) return `Hallo ${firstName},`;
            if (gender === 'm') return `Hallo Herr ${name},`;
            if (gender === 'f') return `Hallo Frau ${name},`;
        } 
        // POLITE: "Guten Tag..." (Simplifies titles usually, but we keep them for safety)
        else if (formality === 'polite') {
            const t = title ? title + ' ' : '';
            if (gender === 'm') return `Guten Tag Herr ${t}${name},`;
            if (gender === 'f') return `Guten Tag Frau ${t}${name},`;
        } 
        // FORMAL: Strict DIN Style
        else {
            const t = title ? title + ' ' : '';
            if (gender === 'm') return `Sehr geehrter Herr ${t}${name},`;
            if (gender === 'f') return `Sehr geehrte Frau ${t}${name},`;
        }
        
        return P.SALUTATION_DEFAULT;
    },

    /**
     * Smart Closing Derivation [DIN-LOGIC-GREETING-2026]
     */
    deriveClosing: function(formality) {
        // We could also check the "Subject" tone here in future (e.g. "Mahnung" vs "Einladung")
        const P = window.App.Config.PHRASES;
        switch (formality) {
            case 'casual': return P.CLOSING_CASUAL; // "Viele Grüße"
            case 'polite': return P.CLOSING_POLITE; // "Beste Grüße"
            case 'formal': default: return P.CLOSING_FORMAL; // "Mit freundlichen Grüßen"
        }
    }

window.App = window.App || {};

window.App.Logger = {
    el: null,
    contentEl: null,
    isCollapsed: true,
    history: [], // RAM Buffer
    MAX_HISTORY: 50,
    STORAGE_KEY: 'din_crash_log',

    init: function () {
        this.loadBlackBox();
        this.render();
        this.attachGlobalHandlers();

        this.checkPreviousCrash();

    /**
     * Load previous session logs from storage
     */
    loadBlackBox: function () {
        try {
            const raw = localStorage.getItem(this.STORAGE_KEY);
            if (raw) {
                this.history = JSON.parse(raw);
            }
        } catch (e) {
            console.error('Failed to load crash log', e);
            this.history = [];
        }
    },

    /**
     * Check if the last session ended with an error
     */
    checkPreviousCrash: function () {
        if (this.history.length > 0) {
            const lastEntry = this.history[this.history.length - 1];
            if (lastEntry.level === 'error') {
                // Delay slightly to ensure UI is ready
                setTimeout(() => {
                    this.error('CRASH_DETECTED', `Previous session ended with error: ${lastEntry.message}`);
                    this.toggle(false); // Auto-open console
                }, 500);

        }
    },

    render: function () {

        document.getElementById('btn-clear-log').addEventListener('click', () => this.clear());
        document.getElementById('btn-export-log').addEventListener('click', () => this.exportLog('csv'));
    },

    toggle: function (forceState) {
        if (typeof forceState === 'boolean') {
            this.isCollapsed = forceState;
        } else {
            this.isCollapsed = !this.isCollapsed;
        }
        this.el.classList.toggle('collapsed', this.isCollapsed);

     * Exports the black box log as a file.
     * @param {'txt'|'csv'} format

        if (format === 'csv') {
            content = "Timestamp,Level,Context,Message\n";
            content += this.history.map(e => {
                const msg = e.message.replace(/"/g, '""'); // CSV escape
                return `"${e.time}","${e.level}","${e.context}","${msg}"`;
            }).join('\n');
            mime = 'text/csv';
            ext = 'csv';

                `[${entry.time}] [${entry.level.toUpperCase()}] [${entry.context}] ${entry.message}`
            ).join('\n');

        const url = URL.createObjectURL(blob);
        const a = document.createElement('a');

        this.history.push(entry);
        if (this.history.length > this.MAX_HISTORY) {
            this.history.shift();
        }
        try {
            localStorage.setItem(this.STORAGE_KEY, JSON.stringify(this.history));
        } catch (e) {
            // Storage likely full, ignore to prevent recursive errors
        }
    },

                <span class="log-context">${context}:</span>
                <span class="log-msg">${message}</span>
            `;
            this.contentEl.appendChild(entryDiv);
            this.contentEl.scrollTop = this.contentEl.scrollHeight;
        }

        // 2. Persist
        this.persist({
            time: timestamp,
            level: level,
            context: context,
            message: message
        });

        // 3. Auto-expand on error
        if (level === 'error' && this.isCollapsed) {
            this.toggle(false);
        }
    },

    log: function (context, message) {
        console.log(`[${context}] ${message}`);
        this.print('info', context, message);
    },

    warn: function (context, message) {
        console.warn(`[${context}] ${message}`);
        this.print('warn', context, message);
    },

    error: function (context, message) {
        console.error(`[${context}] ${message}`);
        this.print('error', context, message);
    },

            this.error('GLOBAL_TRAP', `${msg} (Line: ${lineNo})`);
            return false;
        };

        window.onunhandledrejection = (event) => {
            this.error('PROMISE_TRAP', `Unhandled Rejection: ${event.reason}`);
        };
    }
};

window.App = window.App || {};

window.App.Utils = {
    
    /**
     * Creates a debounced function that delays invoking func until after wait milliseconds.
     */
    debounce: function(func, wait) {
        let timeout;
        return function(...args) {
            const context = this;
            clearTimeout(timeout);
            timeout = setTimeout(() => func.apply(context, args), wait);
        };
    },

    /**
     * Async sleep helper for backoff strategies.
     */
    sleep: function(ms) {
        return new Promise(resolve => setTimeout(resolve, ms));
    },

    /**
     * Generates a pseudo-UUID.
     */
    uuid: function() {
        return Date.now().toString(36) + Math.random().toString(36).substr(2);
    },

    /**
     * Deep clones a JSON-compatible object.
     */
    deepClone: function(obj) {
        return JSON.parse(JSON.stringify(obj));
    },

    /**
     * Escapes HTML to prevent basic XSS in innerHTML injections
     */
    escapeHtml: function(str) {
        if (!str) return '';
        return str
            .replace(/&/g, "&amp;")
            .replace(/</g, "&lt;")
            .replace(/>/g, "&gt;")
            .replace(/"/g, "&quot;")
            .replace(/'/g, "&#039;");
    }
};

## 1. System Meta-Rules & Logic

### Address Autocomplete [DIN-UI-AUTOCOMPLETE]

- **Trigger**: Starts after 3 characters are typed in the Recipient field.
- **Sources**:
  1. **Local Contacts**: Exact or partial matches from `App.Storage` (marked with ⭐).
  2. **Remote API**: Suggestions from Photon/Geoapify.
- **Prioritization**: Local contacts ALWAYS appear above remote results.
- **Bias**: Remote searches are geographically biased towards the sender's location (Home/Profile coords) to prevent irrelevant results (e.g. "Berlin, Ohio" vs "Berlin, Germany").
- **Interaction**: Selecting a result replaces the entire recipient field content and triggers Smart Greeting logic.

### Address Validation [DIN-LOGIC-VAL-ADDR]

- **Goal**: Informative feedback on address quality.
- **Rules (DACH region)**:
  - ZIP: Must contain 4 or 5 consecutive digits.
  - Street: Must contain a name and a number (e.g., "Musterweg 12").
  - Sender specific: Should contain at least one communication channel (Email or Phone).
- **Severity**: "Warning" (Orange), never "Error" (Red). System must remain usable.

### Debugging & Error Handling [DIN-SYS-ROBUST]

- **Naming Convention**: System-Flags werden IMMER groß geschrieben (z.B. `DEBUG`).
- **Error Trapping**: Jede Funktion, die I/O betreibt (Storage, Fetch) oder DOM-Events verarbeitet, MUSS in einen `try-catch` Block gehüllt sein.
- **Reporting**: Fehler werden primär an `App.Logger.error` gemeldet. Der Logger entscheidet über die Anzeige (Auto-Expand der Konsole bei Error).
- **Sanitization**: Bei Storage-Fehlern erfolgt ein automatischer Fallback auf `App.Storage.getDefaultSchema()`.

### Core Utilities [DIN-SYS-UTILS]

- **Scope**: Hilfsfunktionen ohne Geschäftslogik (UUID, Debounce, Escaping).
- **Pure Functions**: Dürfen keine externen Abhängigkeiten (DOM/Storage) haben.

### Flight Recorder [DIN-SYS-LOGGER]

- **Black Box**: Zeichnet die letzten 50 Aktionen im RAM auf.
- **Persistence**: Speichert Fehlerzustände im LocalStorage für Diagnosen nach Reload.
- **Privacy**: Keine Übermittlung an externe Server.

### Panic Mode [DIN-SYS-PANIC]

- **Trigger**: Kritische Initialisierungsfehler (White Screen of Death).
- **Fallback UI**: Minimales Overlay zum Exportieren der Daten (Not-Rettung).

### Safety Features [DIN-SAFETY-PREFLIGHT] [DIN-SAFETY-BLACKBOX]

- **Pre-Flight**: Prüft Module (Core, UI, etc.), DOM (#sheet), und Storage-RW. Blockiert App bei Fehler.
- **Black-Box**: Speichert letzte 50 Logs. Prüft beim Start auf vorangegangenen Crash ('error' level).

### Smart Greetings [DIN-LOGIC-GREETING-2026]

- **Parsing**: Analyse des Empfängerfeldes auf Geschlecht (Herr/Frau) und Titel (Dr./Prof.).
- **Formality Cascade**:
  - *Formal*: "Sehr geehrter Herr Dr. Müller," (Voller Titel).
  - *Polite*: "Guten Tag Herr Müller," (Vereinfacht, "Tag" statt "Sehr geehrte").
  - *Casual*: "Hallo Max," (Vorname bevorzugt) oder "Hallo Herr Müller,".
- **Fallback**: "Sehr geehrte Damen und Herren," bei Firmenadressen oder Uneindeutigkeit.

### Sender Profile [DIN-FEAT-PROFILE]

- **Fields**: Company (Optional), Name, Street, Zip, City, Phone, Email, IBAN.
- **Rules**: Firmenname verdrängt Personennamen in Zeile 1 des Absenderblocks, falls vorhanden.
- **Formatting**: Absender-Block nutzt Zeilenumbruch als Trenner.

### Date Formatting [DIN-LOGIC-DATE-FORMAT]

- **Strictness**: Dates MUST follow `DD.MM.YYYY` (DIN 5008 numeric standard).
- **Padding**: Leading zeros are mandatory (e.g., `01.05.2024` not `1.5.2024`).
- **Trigger**: Logic enforces this on `blur` events of the date field.

# 📏 DIN 5008:2020 Domain Knowledge (The Bible)

> **File:** `.brain/03_domain_knowledge.md`
> **Role:** Dictionary of DIN Standards & Validation Rules.
> **Dependencies:** Implemented by `css/din5008-paper.css`, `js/logic/formatter.js`.
> **Description:** Defines physical millimeters, font sizes, margins, and content rules. This file is the SUPREME SOURCE OF TRUTH.

---

## 1. Global Page Dimensions [DIN-PAGE]
- **Paper Format**: DIN A4 (210mm x 297mm).
- **Margins**: 
  - Left: 25mm (min) - Fluchtlinie.
  - Right: 20mm (min).
  - Bottom: Standard text area ends.

## 2. Header Variations [DIN-HEAD]
- **Form A**: Header height 27mm. Address starts 32mm from top. Fold mark 1 at 87mm.
- **Form B**: Header height 45mm. Address starts 50mm from top. Fold mark 1 at 105mm.
- **Punch Mark (Lochmarke)**: Center at 148.5mm.

## 3. The Address Field [DIN-ADDR]
- **Dimensions**: 85mm x 45mm total area.
- **Position**: 20mm from left edge.
- **Internal Structure**:
  - **Zone 1 (Return/Notes)**: Top 17.7mm (5 Lines).
    - *Return Address (Rücksendeangabe)*: Topmost line. Font size 6pt-8pt.
    - *Notes (Vermerke)*: E.g., "Einschreiben", "Nicht nachsenden".
  - **Zone 2 (Address)**: Bottom 27.3mm (6 Lines).
    - *Font Size*: Min 8pt (below 10pt sans-serif recommended).
    - *Constraint*: No empty lines allowed within the address block.
- **International Rules**:
  - Country Name: LAST LINE in CAPS (German spelling, e.g., "BELGIEN").
  - City Name: CAPS (Local spelling, e.g., "LIEGE", "FIRENZE").
  - No country codes (e.g., no "B-" or "I-").

## 4. The Info Block [DIN-REF]
- **Position**: 
  - Vertical: Starts at 32mm (Form A) or 50mm (Form B).
  - Horizontal: 125mm from left edge.
- **Width**: Max 75mm.
- **Content Order**: Ihr Zeichen, Ihre Nachricht vom, Unser Zeichen, Unsere Nachricht vom, Name, Telefon, E-Mail, Datum.
- **Note**: Replaces the old "Bezugszeichenzeile".

## 5. Content Structure & Layout [DIN-BODY]

### A. Subject Line (Betreff)
- **Position**: 2 lines below the address field (not the phantom box, but the content).
- **Prefix**: The words "Betreff", "Betr.", "Betrifft" are **FORBIDDEN**.
- **Syntax**: No period at the end. Multi-line allowed.
- **Highlighting**: Bold is permitted/recommended.

### B. Salutation (Anrede)
- **Position**: 2 lines below the Subject.
- **Separation**: 1 blank line after Salutation before Body Text.

### C. Body Text
- **Line Spacing**: Single line (1.0).
- **Paragraphs**: Separated by 1 blank line.

### D. Greeting (Gruß)
- **Position**: 1 blank line after Body Text.
- **Format**: No comma necessary (but common usage often adds it).
- **Variations**: "Mit freundlichen Grüßen", "Freundliche Grüße nach Hamburg", etc.

### E. Signature & Company
- **Company Name**: 1 blank line after Greeting.
- **Handwritten Sig**: Approx 3 blank lines.
- **Typed Name**: Repeated below the handwritten signature.
- **Additions**: `i. A.` or `ppa.` placed before the typed name.

### F. Attachments (Anlagen)
- **Position**: 
  - Standard: 1 blank line after Typed Name.
  - No Signature: 3 blank lines after Text.
  - Overflow: Right side of Greeting (125mm from left).
- **Syntax**: "Anlage" or "Anlagen". **Bold** allowed. No colon (:).

## 6. Typography & Formatting Rules [DIN-TYPO]

### A. Numbers & Money
- **Grouping**: Space as separator for thousands (e.g., `1 500`, `50 000 000`).
- **Currency**: `25,36 €` or `1.573,45 EUR`. Note: Currency codes use dot separators in strict finance context, spaces in text.
- **Post Office Box**: Grouped from right in pairs (e.g., `3 22 63`).
- **Postcode (PLZ)**: Never grouped (e.g., `12345`).

### B. Dates
- **ISO 8601**: `YYYY-MM-DD` (e.g., `2026-01-09`). Preferred.
- **Numeric**: `DD.MM.YYYY` (e.g., `09.01.2026`). Leading zeros **MANDATORY**.
- **Alphanumeric**: `9. Januar 2026` (No leading zero for day).

### C. Phone Numbers
- **Standard**: Space between Area Code and Number (`030 123456`).
- **Extension**: Hyphen separator (`030 123456-12`).
- **International**: `+49 30 123456` (Drop the leading zero of area code).

### D. Abbreviations
- **Rule 1**: If pronounced as full words, use period + space (`z. B.`, `u. a.`, `i. A.`).
- **Rule 2**: If pronounced as letters/acronym, no points/spaces (`GmbH`, `AG`, `Pkw`, `ZDF`).
- **Units**: Space between number and unit (`10 kg`, `5 m`).
- **Exception**: `usw.` and `usf.` (One point, no space).

### E. Highlighting
- **Allowed**: Bold, Size change, Centering, Indent.
- **Discouraged**: Underlining (cuts descenders like g, p, q).
- **Forbidden**: Spacing out letters (S p e r r e n).

### F. Page Numbering
- **Format 1**: `- 2 -` (Header Center).
- **Format 2**: `Seite 2 von 3` (Footer Right).

## 7. Academic & Professional Titles
- **Placement**:
  - Degrees (Dr., Prof.) -> Before Name (`Herrn Prof. Dr. Müller`).
  - Grads (B.A., M.Sc.) -> After Name (`Max Müller M. A.`).
  - Functions (Director) -> Line below name (`Heinz Schmidt`, `Vorstand`).
